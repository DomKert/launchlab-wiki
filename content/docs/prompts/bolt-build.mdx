## Why this works

The Bolt prompt has a lot of moving parts: project name, palette, type, vibe keyword, hero copy, CTA destination, hero image plan, plus 3 reference screenshots to attach. Filling all of that into a static template by hand means missing one and hitting a wall mid-paste.

Claude reads your Project knowledge (4-box, customer, one-liner, OMTM) and walks you through the missing pieces one at a time. If you're missing a prereq (no style spec, no Tally form, no headline yet), Claude stops and routes you back to the right Spellbook prompt or tool. Then it outputs the final Bolt prompt as one code block you copy straight into Bolt.

This runs in Claude Project, not Bolt. The OUTPUT goes to Bolt.

## Before you run this

Five things ready in your Project knowledge or available to paste:

1. Cleaned section-by-section copy from [Prompt 4](/docs/prompts/landing-page-spec) → [Prompt 9](/docs/prompts/strip-ai-smell). Save as `cleaned-copy.md` in Project knowledge or be ready to paste.

2. Style spec from [Prompt 16](/docs/prompts/visual-direction). Save as `style-spec.md` or be ready to paste.

3. CTA destination set up. [Tally form](https://tally.so) for waitlist, [Cal.com](https://cal.com) for demos, [Stripe Payment Link](https://stripe.com/payments/payment-links) for e-com, mailto: for 1:1. You need the actual URL or email.

4. Hero image plan decided. Either a URL from [Gemini Nano Banana](https://gemini.google.com) or [gpt-image-2](https://chatgpt.com), or "use a placeholder for now", or "type-only, no image."

5. The 3 reference screenshots from Prompt 16 saved to a folder you can find. You'll attach them to Bolt after Claude assembles the prompt.

If any of these are missing, Claude will stop and tell you which to do first.

## Prompt

Run this in your Claude Project chat. Claude walks you through 6 inputs one at a time, then outputs the final Bolt prompt as one code block.

```txt
I'm ready to build my landing page in Bolt. Read my Project knowledge (4-box.md, customer.md, one-liner.md, OMTM, plus cleaned-copy.md and style-spec.md if I've saved them) and synthesize the single Bolt build prompt I'll paste as Bolt's first message.

Walk me through these inputs one at a time, in order. Don't ask all at once. Pull from Project knowledge first if the answer is already there. Only ask when the file isn't saved.

1. Project name (the brand or product name as it appears on the site)
2. Cleaned hero copy from Prompt 9: headline, subhead, CTA button text
3. Style spec from Prompt 16: palette hex codes, type pairing, radius, shadow, spacing, vibe keyword, 3 mood adjectives
4. CTA destination URL: Tally form for waitlist, Cal.com link for demo, Stripe Payment Link for e-com, mailto: for 1:1
5. Hero image plan: paste a URL from Gemini Nano Banana or gpt-image-2 or Unsplash, OR say "placeholder", OR say "no image"
6. Confirm I have 3 reference screenshots saved to a folder ready to attach to Bolt

If I'm missing a prereq (no style spec because I haven't run Prompt 16, no CTA URL because I haven't set up Tally, no headline because Prompt 9 isn't done), STOP. Tell me which Spellbook prompt or tool I need to complete first. Don't fabricate values, don't move on without them.

When you have all six, output the final Bolt build prompt as ONE code block I can copy and paste. Use this exact structure with my values substituted:

---

[I will attach 3 reference screenshots via the paperclip in Bolt]

Build the HERO SECTION ONLY of a single-page landing page for [PROJECT NAME] as a Next.js + Tailwind app. Mobile-first, deploy-ready. Don't build the rest of the page yet, I'll add sections one at a time after I approve the hero.

Vibe keyword: [vibe keyword from style spec]

Brand
Palette: bg [hex], ink [hex], accent [hex], muted [hex]. Use bg as page background, ink for primary text, accent for ONE primary CTA only.
Type: headings in [font], body in [font]. [one sentence on weight and contrast from style spec]
Radius: cards and buttons [px], images [px]
Shadows: [from style spec]
Spacing: section vertical padding [px], max content width [px]
Mood: [3 adjectives from style spec]

Reference vibes
3 attached screenshots are pages whose FEEL I want to match. Don't copy their content. Copy their visual rhythm, density, type contrast, and color logic.

Hero copy (use verbatim, don't rewrite)
Headline: [headline]
Subhead: [subhead]
CTA button text: [CTA text]
CTA destination: button posts to [URL or mailto:]

Hero image
[Write the founder's choice as ONE line: a URL line, OR a placeholder line, OR a "type-only hero with strong color block on the right 30%" line. Delete the other options.]

Constraints
- No corporate stock photo people
- No "Trusted by 10,000+" social proof I haven't given you
- No floating UI cards in the hero
- Mobile must look right first, desktop second
- Single page, no nav

Build the hero only. Stop after.

---

Voice rules for the assembled prompt: don't add anything that's not in my Project knowledge or that I haven't explicitly given you. No fabricated stats, no extra sections, no improvised copy. If I haven't given you a value, ask for it instead of guessing.
```

## After Claude outputs the Bolt prompt

1. Open [bolt.new](https://bolt.new). Pick a pre-loaded Design System on the homepage card before you start (free tier supports the pre-loaded ones, custom design systems need Team plan).
2. Click the paperclip and attach your 3 reference screenshots (10MB limit, free tier).
3. Paste Claude's assembled prompt as your first message.
4. Wait 60 to 90 seconds for the hero scaffold.
5. Don't prompt yet. Read it, click around, decide what's actually wrong vs what just needs a tweak.

For copy edits, color tweaks, padding, hero image swaps: open the file, edit the line, save. Zero tokens.

For new sections (problem block, what-you-offer, CTA, social proof): one prompt per section, referencing the established hero style. "Add the next section using the same palette, type, and radius as the hero. Section content: [paste cleaned copy from Prompt 9]."

Full vibecoding loop and the keep / change / add pattern: [vibecoding workflow](/docs/vibecoding/workflow).

## Need a hero image first?

Use [Prompt 18 Hero image prompt](/docs/prompts/hero-image-prompt) to synthesize the image-gen prompt in Claude (same pattern as this one), then paste into Gemini Nano Banana, gpt-image-2, or Ideogram. Save the URL, drop it in when this synthesizer asks for input 5.

## What good looks like

One Claude Project chat session, 5 to 6 turns, final assembled Bolt prompt in one code block. Paste into Bolt with screenshots attached, hero scaffold in 90 seconds. 3 to 5 follow-up Bolt prompts max for the rest of the page. The rest is direct code edits per [vibecoding workflow](/docs/vibecoding/workflow). Live URL in #launches by Friday night, token budget under 200K.

## What bad looks like

Skipping the synthesizer chat and trying to fill the Bolt template by hand. You miss a placeholder, paste broken syntax, Bolt scaffolds something off, and you spend 30 prompts trying to fix what should have been one prompt.

Or showing up to the synthesizer with no style spec, no Tally form, no cleaned copy. The chat ends in turn 1 telling you to do upstream work first. Bring the prereqs, then run this.
