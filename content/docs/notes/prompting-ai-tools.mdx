The [Spellbook](/docs/prompts) ships 17 canonical prompts. Each one is tested, voice-tuned, and shaped so the next prompt in the chain still works. This note is the layer underneath: the universal rules that apply when you go off-Spellbook, plus the tool-specific flavor that decides which AI to reach for.

If a Spellbook prompt covers your job, use it verbatim. This note is for everything else.

---

## The 6 universal rules

### 1. One chat = one job

Don't reuse a draft chat for editing. Fresh chats catch more. The AI inherits its own bias from the previous turns and stops noticing the words it just used. Run [Prompt 4 landing spec](/docs/prompts/landing-page-spec) in one chat, then open a NEW chat for [Prompt 9 strip](/docs/prompts/strip-ai-smell). Same for copy iterations, research synthesis, anything where you want a fresh read.

### 2. References beat descriptions

For visual work, 3 screenshots will out-direct 100 adjectives. For copy, a sample paragraph in the voice you want will out-direct "write punchy". For research, a competitor URL will out-direct "find similar companies". Always upload, always paste, always link. Description-only briefs produce generic output every time.

### 3. Output contract before prose

Tell the AI the exact shape of what you want before you tell it what to do. "Output 5 bullets, 8 words each, in a code block I can paste." Vague briefs get vague output. The Spellbook prompts all have output contracts baked in. When you write your own, lead with the shape.

### 4. Voice rules upfront

Lead with what NOT to write. "No corporate verbs, no colons-then-lists, no marketing buzzwords" works harder than "write conversationally". Banning specific words and patterns gives the AI something concrete to filter against. See [Writing that doesn't sound AI](/docs/notes/writing-that-doesnt-sound-AI) for the full ban list.

### 5. Hero-first, sections after

In Bolt and any visual builder, never ask for the whole page in one prompt. Build the hero, approve it, then add ONE section per follow-up referencing the established hero style. The full pattern lives in [Prompt 17 Bolt build](/docs/prompts/bolt-build).

### 6. One concern per follow-up

Never stack changes. "Change the hero color, add a new section, fix the CTA" produces a mess where you can't tell what changed. Each prompt does ONE thing. The keep / change / add pattern is in [Vibecoding Principle 6](/docs/vibecoding/principles).

## When to stop and restart

Three prompts produce the same wrong output. Look upstream. Your brief is wrong, not the AI. Open a new chat with a tighter brief, or step back to the upstream Spellbook prompt that feeds this one.

You've spent 30+ minutes on one prompt. Scrap it. Either pull a Spellbook prompt or rewrite from scratch. Time-on-prompt past 30 minutes means the spec is unclear or the tool is wrong for this job.

You're in Bolt and your token budget is bleeding. Escalate down the ladder before prompting again:

1. Direct code edit in the file panel (zero tokens)
2. One-line prompt
3. Multi-line prompt
4. Fresh chat or new project

## Use the Spellbook verbatim

The 17 numbered prompts in the [Spellbook](/docs/prompts) are tested for downstream compatibility. Output shapes, voice rules, and bracketed placeholders are designed so the next prompt in the chain still works.

Don't rewrite, condense, or "improve" them. Replace the bracketed placeholders ([CUSTOMER SENTENCE], [PASTE 4-box.md], etc.) with your actual content. Run the prompt as written.

Ad-hoc rewrites break the chain. Wrong output shapes, em dashes that the strip-AI-smell hook flags, missing fields the next prompt expects. The Spellbook is short on purpose. If you find yourself rewriting one, you probably want a different prompt, not a "better" version of this one.
