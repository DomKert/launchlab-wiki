## Before you start

Open these in tabs.

1. The assembled Bolt prompt from [Prompt 17](/docs/prompts/bolt-build) (one code block Claude synthesized in your Project chat).
2. Your folder with 3 reference screenshots from Prompt 16 + your hero image file from Prompt 18 if you generated one.
3. [bolt.new](https://bolt.new), signed in.
4. Your Claude Project tab kept open so you can feed it back follow-up section prompts without burning Bolt tokens on planning.

## The 6-step sequence

### 1. Pre-build setup. Design System + Plan mode + attach files.

On the [bolt.new](https://bolt.new) homepage, pick a Design System from the dropdown. Default is Shadcn for most shapes (waitlist, demo, e-com, newsletter, 1:1). See [Prompt 17 Bolt build prompt](/docs/prompts/bolt-build) for the per-shape map.

Turn on Plan mode using the toggle next to the Build now button. Plan mode makes Bolt write the build plan first that you review and edit before code generates. Cheapest place to catch bad assumptions.

Click the paperclip and attach your 3 reference screenshots from Prompt 16, plus your hero image file from Prompt 18 if you have one (10MB per file).

### 2. First prompt. Paste the assembled prompt from Prompt 17.

One message. The full assembled prompt that Claude synthesized in [Prompt 17 Bolt build prompt](/docs/prompts/bolt-build). Don't summarize, don't paraphrase, don't fill in placeholders by hand. Claude already did that.

Bolt writes the plan in 30 to 60 seconds. Review it. Fix anything off (wrong stack, missed CTA wiring, no mobile, imagined extra sections) directly in the plan. Then click Build.

### 3. Review the scaffold

You'll get a working page in 60 to 90 seconds. Don't prompt yet. Read it. Click around. Check what's actually wrong vs. what just needs a tweak.

### 4. Tweak via code edits, not prompts

For copy edits, hero image swaps, color tweaks, padding changes: open the file, edit the line, save. Zero tokens.

For one-line code changes (rename a variable, change a className): same. Edit directly.

### 5. Re-prompt only for restructures

If you need a new section, a different layout, or a feature that touches multiple files, prompt Bolt. One change per prompt. Use the keep / change / add pattern from [Principle 6](/docs/vibecoding/principles).

### 6. Publish

Bolt's Publish button is one click. Your URL ends in `.bolt.host` (or you can connect a custom domain on a paid plan). For free tier, the bolt.host subdomain is fine for the weekend launch.

Drop the URL in #launches before tonight's check-in.

## What good looks like

A 60-min build. One initial prompt. 3 to 5 follow-up prompts max. The rest is code edits. Token budget under 200K. Live URL in #launches by Friday night.

## What bad looks like

A 4-hour build. 30+ prompts. Every fix breaks two other things. Token budget gone before the page is even live. Migration to local on Day 2 instead of Day 1.
